## 2. Components Requiring Enhancement

### 2.1 Event-Specific Diet Plans (HIGH PRIORITY)

**Current State:** Basic wedding week / pre-wedding / post-wedding phases implemented.

**Required Enhancement:**
- Add event-specific meal plans for:
  - Haldi Ceremony - anti-inflammatory, yellow-food theme
  - Mehendi Night - light snacks for long sitting
  - Sangeet/Garba Night - high-energy pre/post dance nutrition
  - Baraat Day - endurance nutrition for dancing
  - Reception - celebration awareness

**Implementation Approach:**
```dart
// Add to wedding_plan_rule.dart
static WeddingDayPlan _eventSpecificPlan(String eventKey, WeddingPlanRole role) {
  switch (eventKey) {
    case 'haldi': return _haldiDayPlan(role);
    case 'mehendi': return _mehendiDayPlan(role);
    case 'sangeet': return _sangeetDayPlan(role);
    case 'baraat': return _baraatDayPlan(role);
    case 'vivah': return _weddingDayPlan(role);
    case 'reception': return _receptionDayPlan(role);
  }
}
```

### 2.2 Meal Planner Integration (HIGH PRIORITY)

**Current State:** WeddingDayPlan provides meal suggestions.

**Required Enhancement:**
- Connect wedding plan to meal planner for one-tap logging

- Wedding event-day meal templates auto-surface

**Implementation:**
- Add `weddingPhase` parameter to meal planner
- Create wedding-aware meal templates
- Enable "Log Today's Wedding Meal" quick action

## 8. Summary

The Wedding Planner module is **~70% complete** based on the v2.0 specification. The core infrastructure (domain models, screens, providers, database schema) is fully implemented. The remaining work focuses on:

1. **Event-specific diet plans** - Adding specialized meal plans for each wedding event
2. **Meal planner integration** - Connecting wedding plans to food logging
3. **Notifications** - Event-day push notifications
4. **Karma integration** - Wedding-specific XP rewards

The wedding module follows the offline-first architecture correctly, stores data in Drift, and integrates with the dashboard. Once the above enhancements are complete, the module will fully align with the v2.0 documentation specification.
